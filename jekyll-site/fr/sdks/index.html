Modes d'intégration

Chaque binding de langage fournit trois classes qui couvrent différents scénarios de déploiement :

Mode Fonctionnement Quand l'utiliser
Report Lance dotnet RdlCmd.dll comme processus enfant via le shell système. Nécessite que le runtime .NET soit installé sur le serveur. Configuration la plus simple — adaptée au développement ou lorsque .NET est déjà présent sur la machine.
ReportAot Lance un binaire compilé en AOT autonome (RdlCmd). Aucun runtime .NET requis — le binaire inclut tout. Déploiements en production où vous ne pouvez pas installer le runtime .NET. Copiez simplement le binaire à côté de votre application.
ReportNative Charge librdlnative.so en processus via FFI native (ctypes / PHP FFI / Fiddle). Aucun processus enfant lancé. Latence minimale, débit maximal. Requis pour le rendu en mémoire sans écrire de fichiers temporaires (Python uniquement pour l'instant).
