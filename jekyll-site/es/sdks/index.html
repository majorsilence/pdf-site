SDKs de lenguajes

Usa Majorsilence Reporting
desde cualquier lenguaje

Los wrappers de Python, PHP y Ruby te permiten renderizar informes RDL a PDF, Excel, CSV, HTML y más, sin escribir ningún código .NET. Cada lenguaje incluye tres modos de integración para adaptarse a tus restricciones de despliegue.

Python
majorsilence-reporting

Python 3.10+. subprocess, binario AOT, o FFI ctypes en proceso. Renderizado real en búfer en memoria en modo nativo.

Modos de integración

Cada binding de lenguaje incluye tres clases que cubren distintos escenarios de despliegue:

Modo Cómo funciona Cuándo usarlo
Report Lanza dotnet RdlCmd.dll como un proceso hijo a través del shell del sistema. Requiere que el runtime de .NET esté instalado en el servidor. Configuración más sencilla: útil para desarrollo o cuando .NET ya está instalado en la máquina.
ReportAot Lanza un binario autocontenido compilado con AOT (RdlCmd). No requiere en absoluto el runtime de .NET: el binario incluye todo. Despliegues de producción donde no puedes instalar el runtime de .NET. Copia el binario junto a tu aplicación.
ReportNative Carga librdlnative.so en proceso mediante FFI nativo (ctypes / PHP FFI / Fiddle). No se genera ningún proceso hijo. Menor latencia, mayor rendimiento. Necesario para renderizado en memoria sin escribir archivos temporales (por ahora, solo en Python).
